Si vous vous intéressez à l'accessibilité web, vous avez rencontré deux acronymes : RGAA et WCAG. Le premier est français, le second international. Cet article clarifie les différences et vous dit lequel appliquer.

## WCAG : la norme internationale

Les WCAG (Web Content Accessibility Guidelines) sont publiées par le W3C. La version en vigueur est la 2.1 (2018), la 2.2 a été finalisée en 2023.

Les WCAG s'organisent autour de quatre principes : perceptible, utilisable, compréhensible, robuste. Trois niveaux de conformité : A (minimum), AA (recommandé), AAA (optimal).

Le niveau AA est celui que la plupart des législations exigent : contraste minimum 4.5:1, alternatives textuelles, navigation clavier, sous-titres vidéo.

## RGAA : le cadre français

Le RGAA (Référentiel Général d'Amélioration de l'Accessibilité), publié par la DINUM, version 4.1, est une **traduction opérationnelle** des WCAG adaptée au contexte français. Il transforme les critères WCAG 2.1 AA en 106 critères testables, regroupés en 13 thématiques.

Là où les WCAG disent "les images informatives doivent avoir une alternative textuelle", le RGAA détaille exactement comment tester ce critère.

## Les différences concrètes

**Portée juridique.** Le RGAA a force de loi en France. Les WCAG n'ont pas de valeur juridique directe en droit français.

**Granularité.** WCAG 2.1 AA = 50 critères de succès. RGAA 4.1 = 106 critères opérationnels plus précis.

**Méthodologie d'audit.** Le RGAA prescrit une méthodologie précise (échantillon, grille, calcul du taux). Les WCAG ne définissent pas de méthodologie.

**Livrables administratifs.** Le cadre français impose : déclaration d'accessibilité, schéma pluriannuel, plan d'action annuel. Ces obligations n'existent pas dans les WCAG.

**Version de base.** Le RGAA 4.1 est basé sur les WCAG 2.1. Les WCAG 2.2 ne sont pas encore intégrées.

## Quel référentiel appliquer ?

**PME en France uniquement** : appliquez le RGAA. C'est le référentiel qui fait foi. Pour les obligations légales, consultez [notre article sur la loi 2025](/blog/accessibilite-web-obligatoire-pme-2025).

**Présence internationale** : visez WCAG 2.1 AA + obligations administratives françaises.

**Prestataire web** : maîtrisez le RGAA pour les clients français, les WCAG pour les clients internationaux.

## Les idées reçues

**"Les overlays rendent conforme."** Faux. Les overlays ne modifient pas le code source. Un audit évalue le code.

**"Les WCAG sont plus strictes."** Faux. Le RGAA est une déclinaison des WCAG AA. Même niveau d'exigence.

**"Si je suis conforme WCAG 2.2, je suis conforme RGAA."** Techniquement oui pour les critères techniques. Mais les obligations administratives restent spécifiques au cadre français.

## La démarche recommandée

Commencez par un pré-diagnostic automatisé. Corrigez les erreurs évidentes — détaillées dans [notre article sur les erreurs courantes](/blog/erreurs-accessibilite-web-courantes-pme). Puis faites réaliser un audit RGAA complet.

## Ce qu'il faut retenir

Le RGAA est la déclinaison française des WCAG, avec 106 critères opérationnels et des obligations administratives spécifiques. Pour une PME en France, c'est le RGAA qui fait référence.
